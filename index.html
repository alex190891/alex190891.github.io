Projects

Implicit Entity Recognition on Twitter

Entity Recognition and Linking is an important task in Natural Language Processing and Text Mining. Applications like Trend Detection, Opinion Mining, Sentiment Analysis and Event Monitioring rely on Named Entity Recognition and Linking. State of the art systems have focussed only on identifying and recognizing explicit entity mentions from unstructured text. However, entities are mentioned implicitly more often than not. This project aims at identifying and linking implicit entity mentions from unstructured text (Tweets).

Detecting and Classifying Implicit Entity Mentions in Tweets (Master's Thesis): My thesis focuses on identifying tweets with implicit entity mentions using background knowledge. State of the art Entity Recognition systems identify explicit entity mentions in tweets. They have never addressed the challenge of identifying entities which are mentioned implicitly in tweets. The exisiting approaches use Conditional Random fields and language level features to identify explicit entity mentions and do not capture the underlying semantics of the tweets. My thesis addresses this novel task of identifying implicit entity mentions in tweets and classifying them into a predefined type. This approach leverages crowd sourced knowledge bases like Wikipedia and DBpedia to capture the entities that are relevant to the domain. We score the relevance of each of these entities using the Wikipedia Hyperlink Graph and Pointwise Mutual Information. We leverage these semantic features along with the syntactic features of tweets to spot the implicit entity mentions and classify them.

Implicit Entity Linking in Tweets: Entity Linking algorithms link named entities in unstructured text to external knowledge base. State of the art entity linking systems link only explicit entity mentions. This work aims at linking implicit entities in tweets that are selected by the Implicit Entity Recognition work by leveraging contextual and factual knowledge extracted from crowd sourced knowledge bases like DBpedia and Tweets.

Real Time Tweet Filtering

Social media has experienced immense growth in recent times. These platforms are becoming increasingly common for information seeking and consumtion. Twitter Streaming API provides a mechanism to stream tweets using keywords. However, this keyword based streaming brings in a lot of irrelevant tweets due to the ambiguous nature of tweets. This work aims to filter out noisy tweets in real time using feedback learning.

I worked on creating the real time analysis pipeline using Apache Storm, Twitter Streaming API and MongoDb. I also developed the framework to classify tweets in real time using Weka, get user feedback and retrain the classification model in real time.

Knowledge Acquisition from Electornic Medical Records

I completed a 3 month intership in the Summer of 2014 at ezDI. ezDI provides products that organize, store and analyze medical data in near real time and embed the results of the analytics into a hospital's clinical workflow. The ezDI Medical Knowledge base is the corner stone for all the applications developed at ezDI(Computer Assisted Coding, Clinical Document Improvement). The knowledge base is rich in hierarchial relationships but lacks coverage in terms of domain relationships. Improving these domain relationships indirectly impacts the performance of all the applications that are dependant on this knowledge base. I used the domain knowledge and semantic and graph techniques to glean meaningful relationships between the concepts of the knowledge base.

eDrugTrends

Over the past several years, synthetic cannabinoids have emerged as a new designer drugs. These drugs, after gaining popularity as legal alternatives to cannabis, have been associated with adverse health effects. This project aims at developing a system for semi-automated processing and visualiztion of thematic, sentiment, spatio-temporal, and social network dimensions of social media data (Twitter and Web Forums) on cannabis and synthetic cannabinoid use. I worked on developing and enhancing the Drug Abuse Ontology (DAO) that is being used by the system.
